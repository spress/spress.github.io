Configuration

Your site have a config.yml file that let you change the default configuration of Spress and create new variables that will be accessible in your template with {{ site.your_variable }}. Some global variables like timezone or safe can be specified in the command line options or flags.

The configuration is YAML file. Do not uses tabs.

Environment configuration: development and production

The environment configuration New in 1.1.0 is useful for writing configuration options for development and production environments. Each Spress site has a config.yml file (mandatory) with the options for the default environment (dev). If you want set options for production environment you can to create a config_prod.yml file with the options that will be overrided in config.yml. The command line option --env="dev" let you to enable the environment.

The pattern for environment configuration file is config_{environment-name}.yml.

An example for production environment:

$ spress site:build --env=prod

More information and example available at this blog post.

Default configuration

Spress runs with the default configuration:

debug: false

# Locations

source: .
destination: ./_site
posts: ./_posts
includes: ./_includes
layouts: ./_layouts
plugins: ./_plugins
include: [.htaccess]

# Reading

env: 'dev'
exclude: [composer.json]
markdown_ext: [markdown,mkd,mkdn,md]
processable_ext: [html,htm,xml,js,css]
layout_ext: [html.twig,twig,html]
permalink: pretty
relative_permalinks: true
drafts: false

# Outputting

url: ''                                 # e.g: http://mydomain.local:4000
paginate: 0
paginate_path: 'page:num'
limit_posts: 0
timezone: null                          # e.g. Europe/Madrid
safe: false

# Serving
host: '0.0.0.0'
port: 4000

# Deprecated:

baseurl: '/'
Name Type Description
include array Force to include files or directories.
exclude array Force to exclude files or directories.
markdown_ext array For Markdown converter, this is a file extension that will be considered as Markdown file.
processable_ext array File extension that will be considered like processable.
drafts boolean Include drafts in the generated site.
safe boolean Disable site plugins.
timezone string Set the Timezone. See more timezones in PHP.

Processable extensions

Processable extension indicates the cadidate files that may be processed.
The final value is the union between processable_ext key and the extension registered by converters.
